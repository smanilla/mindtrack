MindTrack: An AI-Assisted Mental Health Journaling and Monitoring Platform

Project Proposal • MERN + AI/NLP • Submission: End of 8th Semester

1) Title & Abstract

Title: MindTrack: An AI-Assisted Mental Health Journaling and Monitoring Platform

Abstract: MindTrack is a MERN-based web platform that enables patients to track mood and mental health through journaling while allowing healthcare providers to monitor trends and risk indicators. Using AI/NLP, the system generates weekly summaries, flags potential crisis signals, and supports guided assessments. It features role-based access, secure authentication, clinician dashboards, and patient self-help insights. The project aims to improve early detection, support clinician efficiency, and increase patient engagement via privacy-preserving design and explainable AI outputs.

2) Background / Problem Statement

  • Mental health services face rising demand and limited clinician time, reducing continuity of monitoring between sessions.
  • Patients struggle with consistent self-tracking; clinicians often lack longitudinal, structured insights.
  • Existing apps are either purely self-help or lack clinical workflows and secure data sharing.
  • Problem: Provide a secure, clinician-integrated, AI-assisted journaling and assessment tool that surfaces risk signals and trends without increasing clinician burden.

3) Objectives

  • Build a secure, role-based web application for patients and doctors.
  • Enable entries, mood tracking, and AI-guided assessments.
  • Generate weekly AI/NLP summaries and highlight mood trends and crisis indicators.
  • Provide clinician dashboards, patient summaries, and report drill-downs.
  • Ensure privacy (JWT auth, HTTPS-ready), least-privilege data model, and explainable outputs.
  • Deliver usable performance and maintainable, testable code with clear documentation.

4) Methodology

Architecture: MERN stack

  • Frontend: React + Vite + React Router; Axios for API access.
  • Backend: Node.js/Express REST APIs; routes for auth, entries, journals, AI assessment/summary; JWT middleware.
  • Database: MongoDB with Mongoose models (User, Entry, Journal, Assessment).
  • AI/NLP: LLM-based analysis for weekly summaries, sentiment/risk indicators, guided assessments (see OPENAI_SETUP.md).
  • Security: JWT bearer auth, CORS, server-side validation, HTTPS-ready deployment.
  • Data: Minimal PHI, separate collections, created-by ownership, timestamps, mood scores, indexes by user.
  • DevOps/Testing: Nodemon + Vite for local dev; Postman tests (docs/TESTING_POSTMAN.md); deployment to cloud or Docker (planned).
  • Ethics/Explainability: Human-readable rationales; disclaimers; export/delete data on request.

5) Expected Outcome

  • Working platform where patients journal and receive AI-generated weekly summaries and visual mood trends.
  • Clinicians view curated patient summaries and reports for session support and triage.
  • Improved engagement and earlier detection of concerning signals through AI flagging.
  • Secure, maintainable codebase with documentation and endpoint tests.
  • Basis for research on efficacy and usability of AI-assisted mental health monitoring.
